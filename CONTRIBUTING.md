# Contributing to LeetCommit

First off — thank you! Every contribution, big or small, makes LeetCommit better for thousands of developers. 🙏

---

## 🐛 Found a Bug?

1. Check [existing issues](https://github.com/Aditya24Kashyap/LeetCommit/issues) first — it may already be reported
2. If not, [open a new issue](https://github.com/Aditya24Kashyap/LeetCommit/issues/new) with:
   - What you expected to happen
   - What actually happened
   - Steps to reproduce
   - Browser version and OS

---

## 💡 Want to Add a Feature?

1. [Open an issue](https://github.com/Aditya24Kashyap/LeetCommit/issues/new) first to discuss it — saves everyone time
2. Once approved, fork and build it
3. Open a PR referencing the issue

---

## 🔧 Development Setup

```bash
# 1. Fork this repo on GitHub, then clone your fork
git clone https://github.com/Aditya24Kashyap/LeetCommit.git
cd LeetCommit

# 2. Load in Chrome
# Go to chrome://extensions/ → Developer Mode ON → Load unpacked → select this folder

# 3. Make your changes in VS Code or any editor

# 4. Reload the extension after changes
# chrome://extensions/ → click the refresh icon on LeetCommit
```

No npm, no build step, no terminal commands needed. It's plain JavaScript.

---

## 📝 PR Guidelines

- Keep PRs focused — one feature or fix per PR
- Write clear commit messages: `fix: SHA conflict on duplicate push` not `fixed stuff`
- Test on both Chrome and Brave before submitting
- Update README.md if you add a new feature

---

## 🎯 Good First Issues

Looking for where to start? Check issues tagged [`good first issue`](https://github.com/Aditya24Kashyap/LeetCommit/issues?q=label%3A%22good+first+issue%22).

Some ideas:
- Add GeeksforGeeks support
- Add Firefox manifest support
- Improve topic extraction accuracy
- Add more language extension mappings
- Write better error messages

---

## Code Style

- Plain vanilla JavaScript — no frameworks, no build tools
- Comment anything non-obvious
- Keep functions small and named clearly
